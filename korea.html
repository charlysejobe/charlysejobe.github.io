Semester Project — Cybersecurity & AI

Static Control Flow Graph (CFG) Recovery from Windows Malware

Jan 2026 — Jun 2026 EURECOM

Context & Challenge

Traditional malware detection relies on signature-based methods, which can be easily bypassed through code obfuscation. This project aimed to develop a classification approach that resists obfuscation by analyzing the structural properties of malware binaries rather than their raw content.

Approach

  1. Binary Analysis: Extracted Control Flow Graphs (CFGs) from 3,850 Windows PE malware samples using angr, a powerful reverse engineering framework.
  2. Graph Representation: Converted each CFG into a graph data structure suitable for machine learning, preserving the structural relationships between code blocks.
  3. GNN Classification: Trained a Graph Isomorphism Network (GIN) using PyTorch Geometric to classify malware into families based solely on their CFG structure.
  4. Optimization: Implemented Python multiprocessing to parallelize the binary analysis step, significantly reducing processing time.

Key Results

  • Data Processing: Successfully extracted and processed 3,850 static Control Flow Graphs (CFGs) across 235 distinct malware families using angr
  • Model Architecture: Designed and trained a 2-layer GIN with Batch Normalization and Global Mean Pooling to handle massive obfuscated graphs without gradient explosion
  • Performance: Achieved a peak test accuracy of 40.39% on a highly complex 235-class classification task, successfully converging training loss from 5.32 to 3.15
  • Resilience: Demonstrated the GIN model's ability to preserve structural density through injective sums, capturing the complex structures of obfuscated malware better than standard GCNs

Semester Project — IoT & Accessibility

Assistive Device for the Visually Impaired

Sep 2024 — Jan 2025 EURECOM

Context & Challenge

Visually impaired individuals face significant challenges navigating their environment. This project aimed to design a low-cost, portable assistive device that could provide real-time spatial feedback using haptic and audio cues.

How It Works

  1. Camera Input: A camera module connected to a Raspberry Pi 4 captures the environment in real-time
  2. Image Processing: Python scripts analyze the camera feed to detect obstacles and estimate distances
  3. Haptic Feedback: Vibration motors provide directional cues — intensity increases as obstacles get closer
  4. Audio Cues: Complementary audio alerts for critical obstacles or navigation instructions

Hardware Setup

  • Raspberry Pi 4 (main processing unit)
  • Pi Camera Module (environment capture)
  • Vibration motors (haptic feedback)
  • Breadboard & GPIO wiring
  • Speaker module (audio feedback)

Data Processing & Automation — EGM

Jun 2025 — Jul 2025 EGM, Sophia Antipolis

Context & Challenge

During my internship at EGM, an R&D engineering company specializing in IoT and sustainable technologies, I worked with large volumes of data generated by environmental sensors such as the eNose gas detector and Pixelo. The existing evaluation process was highly manual and time-consuming: engineers had to extract data, compute metrics, and format reports by hand.

What I Built

To solve this bottleneck, I designed and deployed an automated Python data pipeline:

  • Modular Data Pipeline: Built a scalable Python architecture (separating configuration parameters from execution scripts) to automatically ingest and parse raw CSV sensor data
  • Data Processing: Utilized the Pandas library to clean, structure, and manipulate the datasets efficiently
  • Automated Visualizations: Integrated Matplotlib to automatically compute key metrics (baselines, normalizations, average curves) and generate formatted charts
  • Report Generation: The script automatically compiles and exports visual reports into designated folders, making them instantly ready for stakeholder review

Lab Work & Hardware Testing

Beyond software development, I also performed hands-on sensor measurements in the laboratory using the eNose and Pixelo devices. This gave me practical experience with IoT hardware, data acquisition protocols, and the full lifecycle from physical measurement to automated analysis.

Impact

The automation tool significantly reduced manual processing time, improved data accuracy by eliminating human errors, and allowed the engineering team to focus entirely on advanced analysis rather than tedious data preparation.
